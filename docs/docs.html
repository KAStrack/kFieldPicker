kFieldPicker

A lightweight, zero-dependency pattern picker for <input> and <textarea> elements. Trigger a searchable dropdown with any character sequence.

Installation

<!-- In <head> -->
<link rel="stylesheet" href="kFieldPicker.css">
<!-- Before </body> -->
<script src="kFieldPicker.js"></script>

CommonJS / ESM

const kFieldPicker = require('kfieldpicker');
// or: import kFieldPicker from 'kfieldpicker';

Quick Start

kFieldPicker('#my-input', {
  trigger: '#',
  items: ['cat', 'dog', 'mouse'],
});

Typing # opens the dropdown. Arrow keys navigate, Enter/Tab selects, Escape closes.

Tip: Accepts a CSS selector, DOM element, or NodeList/array of elements.

Item Formats

// Plain strings
items: ['cat', 'dog']

// Objects
items: [
  { label: 'First Name', value: 'first_name' },
  { label: 'Email', value: 'email', description: 'Primary email address' },
]

Grouped Items

Items can be organised into groups using { group, children } objects. Groups can be nested to any depth — children can themselves contain more group objects. Group headers are rendered as non-selectable section labels and are skipped during keyboard navigation. maxResults counts leaf items only.

items: [
  {
    group: 'Contact',
    children: [
      {
        group: 'Personal',          // nested group
        children: [
          { label: 'First Name', value: 'first_name' },
          { label: 'Email',      value: 'email' },
        ],
      },
      { label: 'Company', value: 'company' }, // leaf at group level
    ],
  },
  {
    group: 'Invoice',
    children: [
      { label: 'Due Date',     value: 'due_date' },
      { label: 'Total Amount', value: 'total_amount' },
    ],
  },
]
Filtering behaviour: When the user types after the trigger, group headers are automatically suppressed if all their children are filtered out. Only groups with at least one matching descendant are shown.
Spaces: A query may contain spaces, so multi-word labels like "Alice Johnson" stay searchable — typing Alice J keeps it shown. A space that matches nothing ends the token and closes the picker (e.g. Alice a). A query can't start with a space. With allowFreeText, spaces are always kept.

Insert Templates

String template

insertTemplate: '{{value}}'  // → {{first_name}}

Function template

insertTemplate: function(item) { return '[' + item.label + '](' + item.value + ')'; }

Without insertTemplate, item.value is inserted. Use includeTriggerInInsert: true to prepend the trigger. Use appendSpace: false to skip the trailing space.

Don't double the trigger: includeTriggerInInsert: true re-adds the trigger character, so your item.value (or insertTemplate output) should not already include it. With trigger: '/', use value: 'close' — not '/close' — otherwise the insert becomes //close.

Free Text

Set allowFreeText: true to let users insert whatever they've typed, even if it isn't in the list. A synthetic row showing the raw query is appended as the last option, marked with a badge (freeTextLabel, default "Add"). It's suppressed when the query already exactly matches a listed item's label or value, so you never get a duplicate.

kFieldPicker('#mentions', {
  trigger: '@',
  items: ['alice', 'bob'],
  allowFreeText: true,
  freeTextLabel: 'Mention',
  onSelect: function(item) {
    if (item.freeText) { /* user typed a value not in the list */ }
  },
});

Because real suggestions are listed first, Enter selects a matching suggestion when one exists; the free-text row is only auto-selected when nothing else matches. The object passed to onSelect carries freeText: true for free-text selections. Inserted text still respects insertTemplate, includeTriggerInInsert, and appendSpace.

Tip: Set freeTextLabel: '' to drop the badge and render the free-text row like a normal item.

Custom Rendering

Pass renderItem(item, query) to control the contents of each row. Return an HTML string (set as the row's innerHTML) or a DOM node (appended). Return a falsy value to fall back to the default row — so you can customise only some rows (e.g. just the free-text row).

renderItem: function(item, query) {
  if (item.freeText) return '➕ Add “' + kFieldPicker.escapeHtml(item.label) + '”';
  var emoji = item.data && item.data.emoji || '•';
  return '<span>' + emoji + ' ' + kFieldPicker.highlight(item.label, query) + '</span>';
}

The item argument is { label, value, description, data, freeText }. The row wrapper (active state, click handling, ARIA, nesting indent) is always managed for you — renderItem only fills the row's content.

Security: a returned string is inserted as HTML. Escape any untrusted text with kFieldPicker.escapeHtml(str), or use kFieldPicker.highlight(text, query) which escapes and wraps the match in <mark class="kfp-highlight">. Return a DOM node to avoid HTML parsing entirely.

Async / Function Items

Set items to a function(query) for fully custom filtering. Return an array (sync) or a Promise resolving to an array; nested group objects are supported. A loading spinner shows while a promise is pending, and stale resolutions are dropped if the user keeps typing.

kFieldPicker('#field', {
  trigger: '#',
  minChars: 1,
  items: function(query) {
    return fetch('/api/search?q=' + encodeURIComponent(query))
      .then(function(r) { return r.json(); }); // → [{label, value}, …]
  },
});

Because the function already filtered, kFieldPicker does not re-filter the results (unlike ajaxUrl) — your matching logic (fuzzy, ranked, remote, whatever) is preserved. maxResults, allowFreeText, grouping, and renderItem still apply.

Debouncing: function items aren't debounced (sync filtering stays instant). If your function does network I/O, debounce/throttle inside it — or use ajaxUrl with ajaxDebounce.

Ajax / Remote Source

Set ajaxUrl to load items from a remote endpoint. The current search term is sent as ?q=<query>, debounced by ajaxDebounce ms.

kFieldPicker('#my-field', {
  ajaxUrl: '/api/fields',
  ajaxDebounce: 250,
  ajaxTransform: function(data) {
    return data.results; // must return a flat or nested items array
  },
});
Client-side filtering: kFieldPicker always applies its own filtering to the result of ajaxTransform. This means you can point ajaxUrl at a static JSON file and filtering will still work. If your server already filters by q, all results will pass through the client-side filter unchanged (every item matches).
CORS: Ensure your API returns Access-Control-Allow-Origin if cross-origin. The ajax example requires a local HTTP server — open file:// paths will block XHR.

Callbacks

kFieldPicker('#my-field', {
  onOpen:      function(query) { },
  onClose:     function() { },
  onSelect:    function(item, insertedText) { /* item: { label, value, description, data } */ },
  onNoResults: function(query) { },
});

Multiple Triggers on One Element

const el = document.getElementById('my-field');
kFieldPicker(el, { trigger: '#', items: ['first_name', 'email'] });
kFieldPicker(el, { trigger: '@', items: ['alice', 'bob'], insertTemplate: '@{value}' });

Theming

All visual values are CSS custom properties. Override globally or scope to a class.

:root {
  --kfp-bg:                #1c1c1e;
  --kfp-item-color-active: #4d9eff;
  --kfp-mark-color:        #4d9eff;
  --kfp-group-color:       #48484a;
}

Built-in dark theme: dropdownClass: 'kfp-dark'

Adapters

An adapter teaches kFieldPicker how to read and write a particular kind of target. The built-in input adapter handles <input>/<textarea>; custom adapters integrate rich-text editors. The target you pass to kFieldPicker() can be a CSS selector, DOM element, NodeList/array, or any opaque object (e.g. an editor instance) that your adapter knows how to drive.

kFieldPicker.registerAdapter('my-editor', {
  getQuery:       function(el, trigger, closeChar) { /* return { query, triggerPos, cursorPos } or null */ },
  insert:         function(el, text, triggerPos, cursorPos) { /* insert text */ },
  getCaretCoords: function(el, pos) { /* return { top, left, bottom } viewport coords */ },
  bind:           function(el, onInput, onKeydown, onReposition) { return function() { /* unbind */ }; },
  getAriaTarget:  function(el) { return el; /* optional: element to mark up as the combobox */ },
});
MethodRequiredNotes
getQuery(el, trigger)yesInspect the caret context; return { query, triggerPos, cursorPos } or null when the trigger isn't active.
insert(el, text, triggerPos, cursorPos)yesReplace the range from triggerPos to cursorPos with text and place the caret after it.
getCaretCoords(el, pos)noReturn viewport { top, left, bottom } for the caret so the dropdown can anchor to it. Omit to anchor to the element.
bind(el, onInput, onKeydown, onReposition)yesWire listeners; return an unbind function. Call onReposition() when the editor scrolls its own content (e.g. an iframe) so the dropdown follows the caret.
getAriaTarget(el)noThe DOM element to receive the combobox ARIA attributes. Defaults to el when it's a DOM node; return null to skip.

TinyMCE adapter

An optional tinymce adapter ships in kFieldPicker.tinymce.js. Load it after kFieldPicker.js and TinyMCE (5+), then attach the picker to the editor instance once it's ready:

<script src="kFieldPicker.js"></script>
<script src="https://cdn.jsdelivr.net/npm/tinymce@6/tinymce.min.js"></script>
<script src="kFieldPicker.tinymce.js"></script>

tinymce.init({
  selector: '#editor',
  setup: function(editor) {
    editor.on('init', function() {
      kFieldPicker(editor, {
        adapter: 'tinymce',
        trigger: '@',
        items: ['alice', 'bob'],
        insertTemplate: '@{value}',
      });
    });
  },
});

It reads the caret from editor.selection, inserts into the active text node, translates caret coordinates from the editor's (possibly iframed) document into top-window space, repositions on editor scroll, and runs its keydown handler ahead of TinyMCE core so Enter/Tab/arrows drive the dropdown instead of the editor.

Iframe ARIA caveat: in classic (iframe) mode the listbox lives in the top document while the editable body lives in the iframe, so aria-controls/aria-activedescendant (cross-document ID references) don't resolve for assistive tech. They resolve normally for inline: true editors. The visual and keyboard experience is unaffected.

Accessibility

kFieldPicker implements the ARIA combobox pattern so screen readers announce the popup and the active option. On initialisation the field — or, for editor adapters, the adapter's ARIA target element — receives:

AttributeOnBehaviour
role="combobox"fieldMarks the input/textarea as controlling a popup. Any pre-existing role is restored on destroy().
aria-autocomplete="list"fieldSuggestions appear in a list as you type.
aria-haspopup="listbox"fieldThe popup is a listbox.
aria-expandedfieldToggled true/false as the dropdown opens and closes.
aria-controlsfieldPoints to the listbox's generated id.
aria-activedescendantfieldTracks the highlighted option's id during keyboard/mouse navigation; cleared on close.
role="listbox"dropdownEach item is a role="option" with a unique id and aria-selected.
Multiple pickers on one element: the single-valued field attributes (aria-expanded, aria-controls, aria-activedescendant) reflect whichever instance acted most recently. Since only one dropdown is open at a time this stays correct in practice, but it's a known limitation of the multi-trigger setup.
